# v0.10.0 Manual Test Plan (App Fixes)

Two layers of testing:

1. **Automated** — `node scripts/verify_changes.js` from the repo root. Checks all data + code invariants (chapter order, tempo, page labels, repeat, heading-only sections, data fixes, settings wiring). Must print `✅ ALL CHECKS PASSED`.
2. **Manual** — this checklist, run against the installed app (`/Applications/Gita Parayana Pace Helper.app`, v0.10.0). Covers what can't be checked headlessly: timing/feel, overlays, the settings panel, persistence.

Launch: the build is signed but **not notarized** — first launch may need **right-click → Open** to clear Gatekeeper. Connect a second display so the projector opens fullscreen there (on a single screen the projector opens as a window that overlaps the operator — expected).

---

## A. Settings panel (new) — `⚙ Settings`

- [ ] Operator window shows a **⚙ Settings** button.
- [ ] Opening it shows fields for: line-end pause (mātrās), Dhyana sloka-end (ms), colophon slowdown (BPM drop), countdown (s), chapter gap (s), and a per-section tempo (SPM) list.
- [ ] Each per-section tempo pre-fills with the chapter's default (Datta 90, Invocation 85, Dhyana 75, Ch1 85, Ch2–18 95, Mahatmyam 95, Samarpana 80; Gita Saram/Arati/Purnam blank).
- [ ] Change **line-end pause** to e.g. `4`, **Save** → play any chapter (not Dhyana): the gap between lines is visibly longer.
- [ ] **Quit and relaunch** the app → the changed values are still there (persisted).
- [ ] **Reset to defaults** → values return to 2 / 30 / 20 / 5 / 3 and section tempos to defaults.
- [ ] Nudge SPM live on a chapter, open Settings, change an unrelated field (e.g. countdown), Save → the live SPM is **not** reset (only an explicit per-section override changes it).

## B. Pacing / pauses

- [ ] Non-Dhyana chapter (e.g. Ch2): pointer pauses ~2 mātrās (one guru beat) at each line end — a natural breath, uniform across line ends.
- [ ] **Dhyana** (Gita Dhyana Shlokas): pacing is its own (not the flat 2-mātrā) — it keeps the meter-aware feel.
- [ ] Colophon page ("|| oṃ tatsaditi …") plays ~5 SPM slower, then returns to normal on the next page.

## C. Display

- [ ] **Asterisks mode**: each line is one contiguous run of ✱ — no wide gaps between word-groups.
- [ ] Invocation Prayers header lines 3 & 4, and Dhyana shloka 14, fit on one line (shrink to fit, no awkward wrap).
- [ ] Page label (operator position bar) shows **"Header"** on title pages and **"Closing"** on the colophon page — never "Shloka 0" / "Shloka 31".

## D. Auto-play flow

- [ ] Autoplay end of any other chapter → folded-hands for the **chapter-gap** seconds, then a **countdown** (with "Listen to Śruti"), then the next chapter starts.
- [ ] Countdown plays in **auto** mode (not just manual).
- [ ] Chapter header: folded-hands/namaskara shows **while** the header animates; no dead pause after the header before the first verse.
